# Cloud-Native Observability Platform

Infrastructure-as-Code observability platform built on Debian 13, Docker Compose, Prometheus, Grafana, and automation tooling.

---

## Overview

The Cloud-Native Observability Platform provides centralized monitoring and visualization for virtual machines and services running in a VMware-based homelab environment.

The project is designed around reproducibility and automation. Infrastructure provisioning, configuration management, runtime deployment, and operational workflows are separated into distinct layers to simplify maintenance and enable rapid rebuilds.

## Architecture

### Monitoring Flow

```text
Git Repository
      ↓
Terraform
      ↓
Ansible
      ↓
Docker Compose
      ↓
monitoring01 (Debian 13)
      ↓
┌─────────────────────────────┐
│ Grafana        :3000        │
│ Prometheus     :9090        │
│ Alertmanager   :9093        │
│ Blackbox       :9115        │
│ n8n            :5678        │
└─────────────────────────────┘
      ↓
Node Exporters
      ↓
Monitored Infrastructure
```

The observability platform follows an Infrastructure-as-Code approach. Terraform provisions infrastructure, Ansible configures hosts, and Docker Compose manages the observability stack. Prometheus collects metrics from Node Exporters running throughout the environment, while Grafana provides centralized dashboards and Alertmanager processes notifications. n8n provides workflow orchestration and automation.

## Components

### Grafana (`:3000`)

Provides dashboards and visualization for infrastructure metrics collected by Prometheus.

### Prometheus (`:9090`)

Collects and stores time-series metrics from Node Exporters and service endpoints.

### Alertmanager (`:9093`)

Processes alerts generated by Prometheus and routes notifications.

### Blackbox Exporter (`:9115`)

Performs endpoint availability and health checks.

### Node Exporter (`:9100`)

Installed on monitored systems to expose CPU, memory, disk, and network metrics.

### Docker Compose

Manages deployment and lifecycle of the observability platform.

### Terraform

Provides reproducible infrastructure provisioning for VMware and AWS environments.

### Ansible

Automates operating system configuration, package installation, firewall configuration, and service deployment.

### n8n

Provides event-driven operational orchestration and GitOps-style workflow automation.

---

## Platform Responsibilities

This project separates infrastructure provisioning, configuration management, runtime deployment, and operational automation into distinct layers.

| Layer          | Tool                 | Responsibility                                                                                     |
| -------------- | -------------------- | -------------------------------------------------------------------------------------------------- |
| Infrastructure | Terraform            | Provisions virtual machines, storage, networking, and firewall rules                               |
| Configuration  | Ansible              | Configures operating systems, installs packages, templates configuration files, and prepares hosts |
| Runtime        | Docker Compose       | Runs Grafana, Prometheus, Alertmanager, Blackbox Exporter, and n8n                                 |
| Automation     | n8n                  | Performs event-driven orchestration and workflow automation                                        |
| Observability  | Grafana / Prometheus | Provides visibility into infrastructure health, metrics, dashboards, and alerting                  |

## n8n Automation Role

n8n acts as the programmable glue for operational workflows.

Instead of embedding complex routing logic inside Alertmanager, Alertmanager can send webhook events to n8n. n8n evaluates conditions, formats notifications, and routes messages to communication platforms or remediation workflows.

### Example Workflow

```text
Prometheus alert fires
        ↓
Alertmanager sends webhook
        ↓
n8n receives alert payload
        ↓
n8n applies routing logic
        ↓
n8n formats notification
        ↓
Discord • Slack • Email • Ticket • Remediation
```

## Quick Start

Clone the repository:

```bash
git clone https://github.com/dj-3dub/monitoring-stack.git
cd monitoring-stack
```

Validate configuration:

```bash
make validate
```

Start the stack:

```bash
make up
```

View status:

```bash
make status
```

View logs:

```bash
make logs
```

## Makefile Targets

### Docker Runtime

| Command         | Description                    |
| --------------- | ------------------------------ |
| `make up`       | Start monitoring stack         |
| `make down`     | Stop stack                     |
| `make restart`  | Restart stack                  |
| `make logs`     | Follow logs                    |
| `make status`   | Show container status          |
| `make ps`       | Show compose services          |
| `make validate` | Validate compose configuration |
| `make update`   | Pull images and restart stack  |

### Automation

| Command                | Description |
| ---------------------- | ----------- |
| `make automation-up`   | Start n8n   |
| `make automation-down` | Stop n8n    |

### Ansible

| Command                      | Description                 |
| ---------------------------- | --------------------------- |
| `make ansible-ping`          | Test host connectivity      |
| `make ansible-check`         | Dry-run monitoring playbook |
| `make ansible-monitoring`    | Configure monitoring server |
| `make ansible-node-exporter` | Configure monitored hosts   |

### Terraform

| Command                  | Description             |
| ------------------------ | ----------------------- |
| `make terraform-init`    | Initialize Terraform    |
| `make terraform-plan`    | Generate execution plan |
| `make terraform-apply`   | Apply changes           |
| `make terraform-destroy` | Destroy infrastructure  |

### Maintenance

| Command       | Description               |
| ------------- | ------------------------- |
| `make backup` | Create backup archive     |
| `make clean`  | Remove stopped containers |
| `make prune`  | Docker cleanup            |
